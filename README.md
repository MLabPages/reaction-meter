# Reaction Meter — 身体反応計測ツール

消費者が製品・サービス（コンセプト、プロトタイプなど）に触れているときの**表情・姿勢・注視**を、スマートフォンやPCのカメラでリアルタイムに計測するWebアプリです。研究のプレ調査やコンセプトテストでの探索的な利用を想定しています。

## プライバシー設計

- 解析はすべて**ブラウザ内**（Google MediaPipe）で実行されます
- **映像・画像は保存も外部送信も一切されません**
- 記録されるのは数値の特徴量（笑顔スコア、姿勢スコアなど）のみです
- ページを閉じるとデータは消えます（CSVでダウンロードしたものだけが残ります）

※ AIモデルのファイル自体はCDN（jsDelivr / Google）から読み込むため、初回はインターネット接続が必要です。

## 起動方法

カメラAPIの制約上、`index.html` をダブルクリックしただけでは動きません。**ローカルサーバー**か**HTTPS**でアクセスする必要があります。

### PCで使う（ローカルサーバー）

このフォルダで以下のいずれかを実行し、表示されたURL（例 `http://localhost:8000`）をChromeで開きます。

```
python -m http.server 8000
```

または

```
npx serve
```

### スマートフォンで使う

スマホのカメラを使うにはHTTPSが必要です。いちばん簡単なのは **GitHub Pages** での公開です。

1. このフォルダをGitHubリポジトリにプッシュ
2. リポジトリの Settings → Pages → Branch を `main` にして保存
3. 発行されたURL（`https://ユーザー名.github.io/リポジトリ名/`）をスマホで開く

公開URLを知っている人はだれでもアクセスできますが、データはその端末内で完結するため、映像や計測データがサーバーに集まることはありません。

## 使い方

1. **カメラ開始** — 初回はモデル読み込みに数秒かかります
2. 参加者IDと刺激ラベル（例: コンセプトA）を入力
3. **記録開始** — 最初の2秒は「基準値の計測」です。参加者に自然な姿勢でいてもらってください（前のめり度・猫背判定の基準になります）
4. 刺激の提示中、注目したい瞬間に**マーク**ボタンでタグ付けできます
5. **記録停止** — 簡易診断が表示され、CSVをダウンロードできます
6. スマホでは **前面/背面切替** で、画面を見る参加者（前面）／実物商品を触る参加者（背面から撮影）の両方に対応できます

## 計測している指標

| CSV列名 | 内容 |
|---|---|
| `smile` | 笑顔の強さ 0–1（口角の上がり） |
| `brow_furrow` | 眉間のしわ 0–1（困惑・不快・集中） |
| `brow_raise` | 眉の上がり 0–1（驚き・興味） |
| `eye_open` | 目の開き 0–1 |
| `jaw_open` | 口の開き 0–1（驚き） |
| `valence` | 感情価 = smile − brow_furrow（+ポジティブ / −ネガティブ） |
| `attention` | 注視フラグ（顔が正面向きで目が開いている = 1） |
| `yaw_proxy` / `pitch_proxy` | 頭部の左右／上下の向きの近似値 |
| `blink_total` | まばたき累計（多い=緊張・負荷、少ない=没入の目安） |
| `smile_events` | 明確な笑顔（smile>0.5）の発生回数 |
| `lean_in` | 前のめり度（基準時点から顔がカメラに近づいた比率。+前のめり / −のけぞり） |
| `shoulder_tilt_deg` | 肩の傾き（度） |
| `head_offset` | 肩の中心に対する頭の左右ずれ |
| `neck_ratio` | 肩幅に対する頭〜肩の距離（低下=猫背傾向） |
| `posture_score` | 姿勢スコア 0–100（傾き・ずれ・猫背から算出） |
| `engagement` | エンゲージメント合成指標 0–1（注視45% + 感情の強さ20% + 前のめり20% + 眉上げ15%） |
| `gaze_x` / `gaze_y` | 視線方向の推定値 −1〜+1（参加者視点で x: +右/−左、y: +下/−上）。虹彩の位置＋頭の向きから算出 |
| `gaze_zone` | 視線ゾーン（3×3グリッド。例 `center-middle`＝中央、`left-up`＝左上） |
| `duchenne` | 目元も動く「本物の笑顔」度 0–1（口角＋頬の持ち上がり。社交辞令的笑顔との区別に） |
| `cheek_squint` | 頬の持ち上がり 0–1 |
| `lip_press` | 口の引き締め 0–1（緊張・感情の抑制） |
| `nose_sneer` | 鼻のしわ 0–1（嫌悪） |
| `mouth_pucker` | 口すぼめ 0–1（懐疑・熟考） |
| `head_move` | 頭の移動速度（正規化座標/秒。高い=そわそわ・落ち着きのなさ） |
| `nod_total` / `shake_total` | うなずき（同意）/ 首振り（否定）の累計回数 |
| `face_touch` | 顔タッチフラグ（手が顔の近くにある=1。思考・迷い・不安のサイン） |
| `marker` | マークボタンで付けたタグ |
| `phase` | `calibration`（基準計測中）/ `main` |

※ 視線は専用アイトラッカーではなくWebカメラからの粗い推定です。3×3ゾーン程度の解像度と考えてください（画面上の細かい注視点の特定には向きません）。詳細指標は「詳細指標を表示（研究者向けオプション）」トグルで確認できます。

時系列CSVは200ms間隔（5Hz）で記録されます。サマリーCSVには記録全体の平均・比率が1行で出力されます。

## 自己報告尺度との関連分析

記録停止後に表示される「自己報告尺度の入力」パネルで、参加者の下位尺度得点（**自動性・活性・安定**。例：1–7件法の項目平均）を入力して保存すると、セッションがブラウザ内（localStorage）に蓄積されます。

- **保存済みセッション一覧** — 参加者×刺激ごとの要約と自己報告得点を一覧表示。**自動性・活性・安定の得点は一覧の表でいつでも直接編集できます**（得点を入れ忘れて保存しても後から追記可能）
- **全セッションCSV** — 1行=1セッションの分析用データ（`selfreport_automaticity` / `selfreport_activation` / `selfreport_stability` 列を含む）。R や Python でそのまま相関・回帰分析に使えます
- **相関ビュー** — 身体反応の要約指標（X）×自己報告（Y）を選ぶと、散布図・回帰直線・Pearson の r をその場で表示。n が10未満の場合は参考値である旨を表示します

セッションデータも数値のみで、この端末のブラウザ内にだけ保存されます。ブラウザのサイトデータを消すと消えるため、**分析用CSVは早めにダウンロードしておいてください**。

## データ回収（協力者の端末から自動送信）

協力者のスマホにCSVを残すと回収漏れが起きやすいため、Googleスプレッドシートへの自動送信機能があります。

### 研究者側の準備（1回だけ）

1. Google スプレッドシートを新規作成する（このシートにデータが溜まります）
2. メニューの「拡張機能 → Apps Script」を開き、[collector.gs](collector.gs) の中身をすべて貼り付けて保存
3. 「デプロイ → 新しいデプロイ → 種類: ウェブアプリ」を選び、
   **実行ユーザー: 自分 / アクセスできるユーザー: 全員** にしてデプロイ
4. 発行されたURL（`https://script.google.com/macros/s/…/exec`）をコピー
5. Reaction Meter の「📤 データ回収設定」にURLを貼り付けて保存 → 「接続テスト」で確認
6. 「📋 設定込みリンクをコピー」で出てくるリンクを協力者に配布。対面の調査では「🔳 QRコードを表示」で出るQRコードを読み取ってもらうのが簡単です

### 協力者へのリンクの渡し方（3通り）

| 方法 | リンクの形 | 向いている場面 |
|---|---|---|
| **既定設定（推奨）** | `…/reaction-meter/`（素のURL） | app.js の `DEFAULT_COLLECTOR` に自分のGAS URLを書いておくと、素のURLを開くだけで送信設定済みになる。主に使う研究者が1人の場合はこれが最短 |
| **短縮コード** | `…/reaction-meter/?c=名前` | app.js の `COLLECTOR_PRESETS` に1行追加すると、`?c=名前` の短いリンクで送信先を切り替えられる。研究室で複数人が使う場合に |
| **フルパラメータ** | `…/reaction-meter/?collector=…`（長い） | ソースコードを編集せずに使える。他の研究室の人に一時的に使ってもらう場合など |

※ 既定設定・短縮コードはソースコード（公開リポジトリ）に送信先URLが載る点に注意。送信先URLを知られると誰でもデータを送れるため、いたずらが心配な場合は `COLLECTOR_TOKEN`（合言葉）を併用してください。

### 複数の研究者が並行して使う場合

データの行き先は「各自が自分のGoogleアカウントでデプロイしたGAS＝自分のスプレッドシート」なので、並行利用してもデータが混ざることは基本的にありません。そのうえで、リンクの配り間違い（例：素のURLを配ってしまい、既定の送信先＝管理者のシートに他の調査のデータが入る）を受信側で防ぐには：

1. **各自が自分の collector.gs の `COLLECTOR_TOKEN` に自分の合言葉を設定する**（例: `"tanaka-2026"`。変更後は「デプロイを管理 → 編集 → 新バージョン → デプロイ」で反映。URLは変わりません）
2. 自分のリンク（`?collector=…&token=合言葉` またはプリセット）に同じ合言葉を含める

こうすると、トークンが一致しない送信は「unauthorized」で**受信側が拒否**するため、宛先違いのデータは混入しません。アプリ側には「トークン不一致」と日本語で表示されます。

※ トークンはリンクやソースコードに載るため、悪意ある送信を防ぐ機密情報ではなく「宛先確認」の仕組みと考えてください。

### 協力者側

配布されたリンクを開くだけで送信設定は完了しています。セッションを保存すると、数値データが自動でスプレッドシートに送信されます（**映像は送信されません**）。送信に失敗した場合はデータが端末に残るので、「未送信セッションを送信」で再送するか、CSVを手動でダウンロードしてもらえば回収できます。

### 回収設定なしで使う場合（共有ボタン）

自動送信を設定していなくても、スマホでは「📤 共有」ボタンからCSVファイルをそのままLINE・メール・AirDropなどで送ってもらえます（Web Share API）。小規模な調査ならこちらだけでも回収できます。

### 補足

- データはシート「summary」（1行=1セッション）と、オプションで「timeseries」（200ms間隔）に追記されます。列はアプリの更新に合わせて自動で増えます
- 保存後に尺度得点を修正すると「未送信」に戻り、再送すると新旧2行がシートに残ります。`saved_at` が同じ行は `received_at` が新しい方を採用してください
- いたずら送信が心配な場合は、collector.gs の `COLLECTOR_TOKEN` とアプリ側の「トークン」に同じ合言葉を設定してください
- URLを知っていれば誰でも送信できる仕組みなので、リンクの配布範囲には注意してください

## 研究利用上の注意

- 表情からの感情推定は**近似**であり、照明・カメラ角度・個人差・文化差の影響を受けます。プレ調査や傾向把握には有用ですが、確証的な結論には自己報告・行動指標との併用（三角測量）を推奨します
- 撮影に先立ち、参加者から**インフォームドコンセント**を取得してください（映像は保存されないことも説明すると同意を得やすくなります）
- 顔検出率（`face_detected`）が低いセッションはデータ品質に注意してください。診断パネルにも警告が出ます
